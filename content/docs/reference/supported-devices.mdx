# Supported Devices

We Love Lights works with Philips Hue and Zigbee-compatible devices connected through supported bridges. This guide lists compatible hardware.

## Supported Bridges

### Philips Hue Bridge

**Hue Bridge v1** (Round)
- Model: BSB001
- Status: Supported
- Limitations: No remote access, older API
- Still functional for local control

**Hue Bridge v2** (Square)
- Model: BSB002
- Status: Fully Supported ✅
- Features: Local + remote access, latest API
- Recommended for all features

**Requirements**:
- Ethernet connection to router
- WiFi network for Mac connectivity
- Power supply included with bridge

### Phoscon deCONZ

**ConBee II USB Stick**
- Status: Supported ✅
- Wider Zigbee device compatibility
- Requires deCONZ gateway software
- Local access only (no remote)

**RaspBee Module**
- Raspberry Pi addon module
- Status: Supported ✅
- Requires deCONZ software
- Advanced user setup

**ConBee III**
- Newer model USB stick
- Status: Should work (Zigbee 3.0 compatible)
- Check with deCONZ compatibility

## Device Limits

### Bridge Capacity

**Philips Hue Bridge**
- Maximum ~50 lights
- Maximum ~12 accessories (sensors, switches, plugs)
- Total ~62 Zigbee devices
- Theoretical Zigbee limit, practical may be lower

**deCONZ Bridge**
- Higher theoretical limits
- Depends on hardware (ConBee vs RaspBee)
- Can support 100+ devices with proper setup

### Recommendations

**Single Bridge**:
- Up to 40 lights: Excellent performance
- 40-50 lights: Good, approaching limit
- 50+ lights: Consider second bridge

**Multiple Bridges**:
- Use when exceeding single bridge capacity
- Better coverage for large homes
- See [Multi-Bridge Setup](/docs/advanced/multi-bridge)

## Compatibility Notes

### Bluetooth-Only Devices

**Not Supported**: ❌
- Bluetooth-only Hue bulbs (no bridge)
- Must use bridge for We Love Lights
- Bridge required for all functionality

### Zigbee 3.0

**Recommended**: ✅
- Zigbee 3.0 devices most compatible
- Backward compatible with Zigbee LL (Light Link)
- Better interoperability

### Brand Mixing

**Generally Works**:
- Mix Philips Hue with third-party Zigbee
- All must connect through same bridge
- Some features may be brand-specific

**Best Practice**:
- Stick to reputable brands
- Check compatibility lists before purchasing
- Test return policy for new devices

## Testing Compatibility

### Before Purchase

1. **Check Bridge Compatibility**:
   - Philips Hue: Check official compatibility list
   - deCONZ: Check Phoscon compatibility database

2. **Look for Zigbee Certification**:
   - Zigbee 3.0 preferred
   - "Works with Philips Hue" badge
   - "Friends of Hue" certification

3. **Read Reviews**:
   - Other users' experiences
   - Compatibility reports
   - Any known issues

### After Purchase

1. **Pair with Bridge**:
   - Follow device instructions
   - Use official Hue app or Phoscon for pairing
   - May require reset if previously paired

2. **Test in We Love Lights**:
   - Should appear automatically after pairing
   - Test all functions (on/off, dim, color, etc.)
   - Check sensor/switch configuration

3. **Report Issues**:
   - If device doesn't work as expected
   - May be bridge limitation vs app limitation
   - [Report issue](/docs/support/reporting-issues) with device model info

## Future Device Support

### New Releases

We Love Lights works through the bridge API, so:
- New Philips Hue devices: Supported when released
- New Zigbee devices: Supported if bridge supports them
- App updates: May add device-specific features

### Discontinuation

Older devices may:
- Continue to work indefinitely (Zigbee standard)
- Lose some features if manufacturer stops support
- Still function for basic operations
